# ArticGrannie - Nettbutikk for Symønstre

## Prosjektbeskrivelse
Målet er å lage en profesjonell nettbutikk for ArticGrannie (min mor) hvor hun kan selge sine unike, håndlagde symønstre (og potensielt strikkemønstre) som digitale PDF-filer. Nettbutikken skal være brukervennlig for kundene og relativt enkel for ArticGrannie å administrere (legge til produkter, skrive blogginnlegg) etter oppsett. Plattformen skal være fleksibel for fremtidig vekst og tilpasning.

**Kjernefunksjonalitet:**
- Salg av digitale PDF-mønstre.
- Betalingsløsning med Vipps og VISA/kort.
- Blogg for SEO og innholdsmarkedsføring.
- Visuelt tiltalende design som reflekterer ArticGrannie-merkevaren.

## Digitalisering av Fysiske Mønstre (Prosess og Veiledning)

Dette er en kritisk prosess for å kunne selge mønstrene digitalt. Målet er å lage PDF-filer av høy kvalitet som kundene enkelt kan printe hjemme (A4/Letter) og eventuelt på et trykkeri (A0/Copyshop-format).

### Nøkkelmål for Digitaliserte Mønstre
*   **Nøyaktighet:** Vektoriserte linjer må være presise og gjenspeile originalmalen eksakt.
*   **Skalerbarhet:** Vektorformat sikrer at mønsteret kan skales opp/ned uten tap av kvalitet.
*   **Kompletthet:** Alle mønsterdeler, markeringer (hakk, trådretning etc.) og instruksjoner må være med.
*   **Brukervennlighet:** Mønstrene skal være enkle for kunden å printe, sette sammen og forstå.

### Utstyr
*   **Scanner:** A4-scanner er et minimum. A3 er ideelt hvis noen mønsterdeler er store, men A4 fungerer (du må da skjøte sammen scans av større deler digitalt).
*   **Godt lys:** Nødvendig hvis du må ta bilder av deler som er for store for scanneren (jevnt, dagslys uten direkte sol er best).
*   **Millimeterpapir:** (Sterkt anbefalt!) Å legge under en gjennomsiktig plastmal ved skanning gir direkte skala-referanse og hjelpelinjer.
*   **Målebånd/Linjal:** For å verifisere mål på de skannede/digitaliserte delene.
*   **Datamaskin:** Med tilstrekkelig ytelse for grafisk programvare.

### Anbefalt Programvare
1.  **Vektorbasert Tegneprogram (essensielt for mønsterdeler):**
    *   **Adobe Illustrator:** Industristandard, kraftig, mange ressurser. Abonnementsbasert.
    *   **Affinity Designer:** Veldig godt alternativ, engangskjøp, profesjonell kvalitet, rimeligere.
    *   **Inkscape:** Gratis og open-source, veldig kapabel. Kan ha en litt brattere læringskurve for noen, men det finnes mange gode tutorials. **Anbefales sterkt å starte med dette pga. kostnad.**
    *   *Valgt programvare for ArticGrannie: `[Fyll inn valgt programvare her]`*
2.  **Bildebehandlingsprogram (for å rense scans/bilder før vektorisering, valgfritt men nyttig):**
    *   **Adobe Photoshop:** Abonnementsbasert.
    *   **Affinity Photo:** Engangskjøp.
    *   **GIMP (GNU Image Manipulation Program):** Gratis og open-source.
3.  **Layout/Publiseringsprogram (for instruksjonshefter):**
    *   **Adobe InDesign:** Profesjonelt for layout. Abonnementsbasert.
    *   **Affinity Publisher:** Godt alternativ, engangskjøp.
    *   **Canva:** (Gratis/Betalt) Kan fungere for enklere instruksjonshefter.
    *   *Vektorprogrammene (Illustrator/Affinity/Inkscape) kan også brukes for layout, men dedikerte DTP-verktøy er ofte bedre for lengre dokumenter med mye tekst og bilder.*
4.  **PDF-leser/-verktøy:**
    *   **Adobe Acrobat Reader (Gratis):** For å se og teste PDF-er.
    *   **Adobe Acrobat Pro (Betalt):** For avansert PDF-redigering og -sammenslåing (ofte er eksportfunksjonene i designprogrammene tilstrekkelige).

### Fremgangsmåte for Digitalisering (Detaljert)
*(Denne seksjonen er en veiledning og ikke en TODO-liste. Se Fase 2 for TODO-punkter knyttet til digitalisering.)*

1.  **Forberedelse av Fysiske Mønstermaler (Plastmaler):**
    *   Rengjør malene grundig.
    *   Sørg for tydelige markeringer (forsterk om nødvendig).
    *   Marker referansepunkter hvis malen er større enn scanneren.
2.  **Skanning eller Fotografering:**
    *   **Scanner (Foretrukket):** Høy oppløsning (300-600 DPI), TIFF/PNG, hvitt ark bak/millimeterpapir under, flatt og skyggefritt, skann store deler i seksjoner med overlapp.
    *   **Fotografering (Alternativ):** Godt kamera, flat bakgrunn, jevn belysning, kamera direkte over, linjal/målebånd i bildet for skala.
3.  **Digital Etterbehandling av Skannede/Fotograferte Bilder (om nødvendig):**
    *   Sett sammen seksjoner, rens opp (juster lys/kontrast, fjern støy), rett opp skjevheter, beskjær.
4.  **Vektorisering (i Illustrator/Inkscape/Affinity Designer):**
    *   Importer bildet, skaler korrekt til 100% (KRITISK!).
    *   Manuell tracing med Pen Tool (anbefalt). Bruk lag for organisering.
5.  **Legge til Digitale Detaljer og Markeringer:**
    *   Sømmonn (Offset Path-funksjon), kornlinje, hakk, tekst (delnavn, størrelse, antall), testfirkant/testlinje (5x5cm eller 10cm linje) for skalaprint.
6.  **Gradering (Sizing - Hvis Mønsteret Skal Komme i Flere Størrelser):**
    *   Avansert prosess. Enten digitaliser hver fysiske størrelse, eller lær graderingsregler i vektorprogrammet. Vurder å starte med én størrelse.
7.  **Layout for Utskrift (PDF-generering):**
    *   **A4/Letter (Print-at-Home):** Arranger deler effektivt, overlappingsmerker, sidenummerering, testfirkant på første side, oversiktsside for sammensetning.
    *   **A0/Copyshop:** Arranger deler på stort format (ofte nestet hvis flere størrelser), testfirkant.
    *   **PDF Eksport:** Vektor-PDF, skarpe linjer, ikke for tykke.
8.  **Lag Instruksjonshefte:**
    *   **Innhold:** Forsidebilde, beskrivelse, størrelsestabell, materialliste, oversikt mønsterdeler, klippeveiledning, steg-for-steg syinstruksjoner med illustrasjoner/bilder, info om sømmonn.
    *   **Design:** Profesjonelt, lettlest. Lagre som PDF.
9.  **Kvalitetskontroll og Testing (ABSOLUTT KRITISK!):**
    *   PRINT SELV (A4/Letter, evt. A0). Sjekk testfirkant, mål, sammensetning.
    *   Sy en Prøvemodell (Toile/Muslin). Sjekk passform, markeringer, instruksjoner.
    *   Få Andre til å Teste (uavhengige testsyere for tilbakemelding).

### Tips for Høy Kvalitet og Effektivitet

## Git Bruk for Dette Prosjektet

Denne `README.md`-filen og eventuelle andre prosjektrelaterte filer (ikke selve WordPress-installasjonen med mindre du gjør avansert tema/plugin-utvikling) kan versjonskontrolleres med Git.

1.  **Initialiser Git i din lokale prosjektmappe (hvor denne README.md-filen ligger):**
    ```bash
    git init
    git branch -M main
    ```
2.  **Legg til og commit filer:**
    ```bash
    git add README.md # Eller git add . for å legge til alle filer i mappen
    git commit -m "Initial commit: Prosjektplan for ArticGrannie nettbutikk"
    ```
3.  **Koble til et GitHub Repository (valgfritt, men anbefalt for backup og samarbeid):**
    *   Opprett et nytt (tomt) repository på GitHub.com (f.eks. `ArticGrannie-ProjectPlan`).
    *   Kjør kommandoene GitHub gir deg for å koble til og pushe, typisk:
        ```bash
        git remote add origin https://github.com/DITT_BRUKERNAVN/DITT_REPO_NAVN.git
        git push -u origin main
        ```
4.  **Videre arbeid:**
    *   Gjør endringer i `README.md` eller andre prosjektfiler.
    *   `git add .`
    *   `git commit -m "En beskrivende melding om endringene"`
    *   `git push origin main` (hvis du bruker GitHub)
